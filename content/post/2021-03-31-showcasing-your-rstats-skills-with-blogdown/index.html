Getting Started with blogdown

First thing‚Äôs first: Install the blogdown package, followed by the static site generator, Hugo.

install.packages("blogdown")
blogdown::install_hugo()
  • We‚Äôll get started with a template Hugo website by creating a new R project (File -> New Project -> New Directory).
  • If you installed blogdown earlier, then in the window that appears after selecting New Directory, you should be able to scroll down and select Website using blogdown - select that.
  • Next, create a directory name (I‚Äôll use my_website), and find a place on your computer to store the directory (I‚Äôll use my Desktop for now, though I‚Äôd recommend a location where it is easy to find!).
  • Keep all boxes checked that are currently checked, and optionally check ‚ÄúOpen in a new session‚Äù (this is a personal preference).
  • Finally, replace the selected ‚ÄúHugo theme‚Äù with: kishaningithub/hugo-creative-portfolio-theme - while there are many themes, including the default, I think the Creative Portfolio will work well with creating a data science portfolio.

Building Your Website

Once your new R project opens, you‚Äôll notice a config.toml file open in the RStudio source editor pane. This is the file where you can modify many of the global settings for your site. Many of these options are fairly straightforward to edit, even if you don‚Äôt know anything about TOML (and I don‚Äôt know anything about TOML üòÑ).

  • Change the title option to something such as your name. This will be the thing that appears in big, bold letters at the top of your website!
  • Add a sentence or two about yourself in the sidebarAbout section. For example, I added "I am an Assistant Professor of Statistics at Simmons University.".
  • You don‚Äôt have to keep all of the options under the [params.social] section. For instance, if you don‚Äôt have an Instagram or choose not to share one on your website, you can just remove the instagram = "full profile url in instagram" section altogether. But for sections that you do want to keep, such as GitHub (for example), you can enter the URL to your GitHub profile page.
  • We‚Äôll keep the three [[params.navlinks]] options for now, but you can change the name of each page if you‚Äôd like, or change which is the ‚Äúhome = true‚Äù page.

At this point, it might be worthwhile to preview your site, just to see what you‚Äôve accomplished so far. To do so, run the following:

blogdown::serve_site()

Note that this does not actually publish your site; it just allows you to view a preview of the site locally in the RStudio ‚ÄúViewer‚Äù pane. You can view your website in the web browser (again, this doesn‚Äôt publish your site) by clicking the icon immediately to the right of the broom in the RStudio ‚ÄúViewer‚Äù pane.

So far, most of what you‚Äôll see came with the template site. And that‚Äôs okay! Because next, we‚Äôll go over how to modify pages within your blogdown website.

Editing Pages

You can edit different sections in your website via the .md files located within the content folder, and you can even add new sections (which you‚Äôd have to specify within your config.toml file that we played with earlier).

For now, we‚Äôll look at the three default sections within the ‚ÄúCreative Portfolio‚Äù Hugo theme: (1) About, (2) Contact, and (3) Home (Portfolio).

About

One of the default pages that comes with the ‚ÄúCreative Portfolio‚Äù Hugo theme is the ‚ÄúAbout‚Äù page. This is where you can describe a little bit more about yourself and your interests, add a photo of yourself, and link to a resume (for example). To access and edit this page within RStudio, go to the ‚ÄúFiles‚Äù pane in the bottom right (unless you reconfigured the RStudio panes) and select the following: content -> about -> _index.md

You can leave the date as is, I don‚Äôt think that matters here. You could change the title to something else if you want - I‚Äôll leave it as About me for now. After that, type whatever you want about yourself!

For example, here‚Äôs what I wrote, taken from the bio in my website (yes, made in blogdown).

Contact

You can access and edit the ‚ÄúContact‚Äù page via content -> contact -> _index.md. Note that this section is called ‚ÄúGet in touch‚Äù on the home page, but you can change the title of this section via config.toml.

This page is pretty straightforward, and there isn‚Äôt much you‚Äôd need to add: maybe an email address, work phone number, or anything else that you‚Äôd feel comfortable sharing that you‚Äôd want folks to use to contact you. This section automatically adds the social media widgets that you specified in config.toml, and it adds an email section that links directly to the email address that you entered earlier.

You might find this section redundant if you kept the email widget in config.toml. You could remove this section altogether, just delete the following in config.toml:

[[params.navlinks]]
  name = "Get in touch"
  url = "contact/"

Portfolio

This is the most significant section of your website, since this is where you can post all the cool stuff you‚Äôve done in R and Data Science! That‚Äôs why it is also the home page by default. Access the Portfolio section via content -> portfolio in the RStudio ‚ÄúFiles‚Äù pane. By default, there will be eleven template .md files in here - you‚Äôll want to move these somewhere else since you don‚Äôt want them to appear in your website. But I wouldn‚Äôt recommend deleting them altogether, in case you want to use one as an example template.

One cool feature that we can use to add a personal touch to the portfolio is the option to add a preview image to any blog post you write, so that the viewer can see the image (along with some preview text) before they read the full article. If you have an image (such as a graph or table) associated with a recent blog post that you‚Äôd like to use, drag the image file into the portfolio folder at the end of this path: static -> img -> portfolio.

Just to start, I dragged the Rmd file associated with a certain blog post of mine into the content -> portfolio folder (where all the .md files were originally saved). I also took a graph from this post and saved it as an image file, and then moved that image to the static -> img -> portfolio folder. Finally, I went into the ‚ÄòYAML‚Äô header of the Rmd file (the content at the beginning sandwiched between the ---) and added this line: image: img/portfolio/nook_bar.png.

All in all, the header in the Rmd blog post file looks like this:

---
title: 'Defeating Tom Nook with #rstats'
author: Anthony Scotina
date: '2020-06-10'
slug: defeating-tom-nook-with-rstats
categories:
  - rstats
  - tidyverse
  - tidytuesday

  - rstats
  - tidyverse
  - tidytuesday

  caption: ''
  image: ''
  preview: yes
image: img/portfolio/nook_bar.png
---
  • Note: Most of those options are automatically populated when you create a new blog post in blogdown (more on that in a sec!).

If you run blogdown::serve_site() again, R will first compile your Rmd file into an HTML, and then it (along with the image) should appear in your website! I repeated this with another post I had previously written (please excuse the shameless self-promotion üôÉ), and this is how they appear in my new website!

New Posts

If you don‚Äôt have any blog posts yet, blogdown makes it real simple to get started! Just select Addins at the top of RStudio‚Äôs source editor pane, and then select New Post. Add a title of your post, the author (that‚Äôs you!), some tags that you might want to appear (I usually add things like ‚Äòrstats‚Äô, ‚Äòtidyverse‚Äô, ‚Äòggplot2‚Äô, etc.), and select the R Markdown format option at the bottom.

Now you‚Äôre all set with writing your first blog post in R Markdown! And if you want to add a preview image afterwards, just drag the image file to the static -> img -> portfolio folder, and add an image: ... option to your YAML header (see my example above).

Deployment

Now that you‚Äôve tweaked the template website in RStudio, and assuming it looks okay when you run blogdown::serve_site(), you‚Äôre ready to publish it online!

In my opinion, the fastest way to deploy your website is through Netlify (https://www.netlify.com/), which provides a free plan to get your website up and running! You can quickly sign up for an account, or log in using your GitHub account.

Once you‚Äôre logged in, click on the Sites tab in Netlify, and then drag the public/ folder built when you run blogdown::serve_site() into the ‚Äúdrag and drop‚Äù space. And that‚Äôs it - your website is built! Note that it will automatically give a URL in the form of random-word-12345.netlify.com, but you can customize this.

You can also make changes to your website in RStudio, and re-deploy to Netlify using this same procedure.
